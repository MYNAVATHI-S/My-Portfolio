Food Share App UI/UX - Step by Step

Step 1: UI Design in Figma

Designed wireframes for user flows: Home, Donate, Request, and Track. Focused on accessibility (large buttons, readable fonts) and minimalism. Colors follow food-safe and trust-inducing tones like green, teal, and orange. Used auto-layout, components, and constraints for responsiveness in Figma.



Step 2: Export Design Assets

Downloaded images, SVG icons (from Figma plugins), button states, and spacing guides. Maintained consistent 8px spacing. Organized assets in a design system with tokens (colors, fonts, sizes).



Step 3: HTML Structure

Built responsive HTML using <header>, <section>, and <footer>. Used divs and flex containers for card layouts. Semantic tags ensured accessibility and SEO. Each screen was built as a section with placeholders for buttons, inputs, and cards.



Step 4: CSS Styling

Used modern styling: Flexbox, Grid for layouts, variables for colors/fonts, and media queries for responsiveness. Shadows, gradients, hover transitions were added to simulate Figma look. Used a dark mode theme by default with consistent padding, margins, and border-radius values.



Step 5: JavaScript Interactions

Implemented step toggle, button animations, scroll-to-top, and interaction logic. Plan to later integrate Firebase or Google Sheets for request form data. Added cursor effects and video controls (like play-on-click) for better interactivity.



Step 6: Testing & Iteration

Tested across devices (mobile, tablet, desktop). Validated responsiveness and accessibility. Adjusted button hit zones, improved readability, and reduced loading time by optimizing video and image assets. Peer feedback helped fix spacing inconsistencies.
